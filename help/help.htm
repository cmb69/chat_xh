Chat_XH - a CMSimple_XH Plugin

Chat_XH facilitates to have an arbitrary amount of simple chat rooms on your website. The user management has to be done with memberpages_XH or preferably register_mod_XH.

Requirements

Chat_XH is a plugin for CMSimple_XH. It requires a UTF-8 encoded version.

Installation

The installation is done as with many other CMSimple_XH plugins. See the CMSimple_XH wiki for further details.

  1. Backup the data on your server.
  2. Unzip the distribution on your computer.
  3. Upload the whole directory chat/ to your server into CMSimple_XH's plugins directory.
  4. Set write permissions to the subdirectories config/, css/, languages/ and the plugin's data folder.
  5. Switch to "Chat" in the back-end to check if all requirements are fulfilled.

Settings

The plugin's configuration is done as with many other CMSimple_XH plugins in the website's back-end. Select "Chat" from "Plugins".

You can change the default settings of Chat_XH in "Plugin Config". Hints for the options will be displayed when hovering over the help icon with your mouse.

Localization is done in "Plugin Language". You can translate the character strings to your own language, or customize them according to your needs.

The look of the Chat_XH can be customized in "Plugin Stylesheet".

Usage

Embedding of a chat room on a page is done with the following plugin call:

{{{PLUGIN:chat('%CHAT_ROOM%');}}}

%CHAT_ROOM% can be choosen arbitrarily, but may contain only alphanumeric character, underscores and hyphens. You can embed as many chat rooms on any single page as you like – they're working independent of each other as long as they have different names.

For automatic displaying of the latest messages JS has to be enabled in the visitors browser. Otherwise the chat is fully usable, but the visitor has to refresh the page manually to see, if new messages have arrived.

To enable user management and to avoid spam, it's recommended to embed chat rooms only on pages, that are protected by register_mod_XH or memberpages_XH. In this case the name of the currently logged in user is stored and displayed along with the message.

After a configurable time of inactivity (i.e. no new messages are posted), the chat history of the chat room will be purged automatically.

If you want to edit the chat data files, you can do so via FTP, or you can configure the data folder to a folder that's accessible for filebrowsers, e.g. userfiles/chat/, and use Ajaxfilemanager_XH.

Note that the chat data files could be accessed directly by calling their URL. A .htaccess file to prohibit this is already contained in the default data folder. For other servers or for custom data folders you have to take for yourself, that those data are protected from non authorized access.
